# Workspace Indexer – Generator

## Overview

Works with any AI agent that supports the [agentskills.io](https://agentskills.io/specification) skill format (Claude Code, Gemini CLI, Copilot CLI, and others).

Solves a common AI-session problem: every time the agent needs to work on a local repo, it scans the entire workspace folder tree to find the path — wasting tokens on a repetitive, mechanical operation.

This skill generates:
1. **`repos.json`** — a pre-built index of every repo in the workspace (`scope`, `name`, `path`)
2. **`refreshworkspace`** — a PowerShell function to regenerate the index when repos change
3. **A `CLAUDE.md` / `GEMINI.md` snippet** — instructs the AI to read the index instead of scanning

## Setup Flow

Ask these questions **before** generating any code. If the user skips a question, use the default.

| # | Question | Default |
|---|----------|---------|
| 1 | **Workspace root** — full path to the folder whose subfolders contain repos (e.g. `D:\Workspace`) | _(required — no default)_ |
| 2 | **Scope subfolder** — name of the intermediate folder between the workspace root and repos (e.g. `Git`, meaning repos live at `<root>\<scope>\Git\<repo>`) | `Git` |
| 3 | **Index file path** — where to write `repos.json` | `<workspace_root>\repos.json` |
| 4 | **Install target** — `profile` (append to `$PROFILE`), `file` (write a `.ps1` file), or `show` (print only) | `show` |

IMPORTANT: All questions above (1–4) are mandatory, and the answers must be provided directly by the user, not assumed, invented, or generated by the agent.

After collecting answers, **generate the index immediately** by running:

```powershell
$root = '<WORKSPACE_ROOT>'
$subDir = '<SCOPE_SUBFOLDER>'
$repos = @()
foreach ($scope in Get-ChildItem -Path $root -Directory) {
    $gitDir = Join-Path $scope.FullName $subDir
    if (Test-Path $gitDir) {
        foreach ($p in Get-ChildItem -Path $gitDir -Directory) {
            $repos += [PSCustomObject]@{ scope = $scope.Name; name = $p.Name; path = $p.FullName }
        }
    }
}
$repos | ConvertTo-Json -Depth 3
```

Show the discovered repo count and first few entries to the user and ask: **"Does this look right? Any scopes to exclude?"**

Then confirm all choices before generating.

## Generated Output

Replace placeholders with the user's answers and emit all three outputs.

### 1. `repos.json` (generated in-session, written to disk)

```json
[
  { "scope": "<SCOPE>", "name": "<REPO_NAME>", "path": "<FULL_PATH>", "description": "<AUTO_DETECTED>" }
]
```

Description is auto-detected in priority order: `package.json` → `description` field, then `README.md` → first non-heading paragraph line, then `""`.

### 2. PowerShell `refreshworkspace` function

```powershell
function refreshworkspace {
    param(
        [string]$WorkspaceRoot = '<WORKSPACE_ROOT>',
        [string]$OutputFile    = '<INDEX_FILE_PATH>'
    )
    $scopes = Get-ChildItem -Path $WorkspaceRoot -Directory
    $repos  = @()
    foreach ($scope in $scopes) {
        $gitDir = Join-Path $scope.FullName '<SCOPE_SUBFOLDER>'
        if (Test-Path $gitDir) {
            foreach ($p in Get-ChildItem -Path $gitDir -Directory) {
                $repos += [PSCustomObject]@{
                    scope = $scope.Name
                    name  = $p.Name
                    path  = $p.FullName
                }
            }
        }
    }
    $repos | ConvertTo-Json -Depth 3 | Set-Content -Path $OutputFile -Encoding UTF8
    Write-Host "Workspace index updated: $($repos.Count) repos → $OutputFile" -ForegroundColor Green
}
```

### 3. CLAUDE.md / GEMINI.md snippet

```markdown
## Workspace Index

A pre-built repo index lives at `<INDEX_FILE_PATH>`. **Always read this file to resolve
repo paths — never scan the workspace folder tree.**

Format:
```json
[
  { "scope": "MyScope", "name": "my-repo", "path": "<FULL_PATH_EXAMPLE>" }
]
```

To find a repo: read `<INDEX_FILE_PATH>`, filter by `name` (or `scope`), use the `path`
field directly. If the file is missing, ask the user to run `refreshworkspace` in their
PowerShell terminal to regenerate it.
```

## Install Steps

**`profile`** — Append to the user's PowerShell profile:
```powershell
Add-Content -Path $PROFILE -Value "`n<generated function>"
. $PROFILE
```
Then run `refreshworkspace` once to generate the index file.

**`file`** — Write to a `.ps1` file and dot-source it from `$PROFILE`:
```powershell
# Add to $PROFILE:
. "C:\path\to\workspace-indexer.ps1"
```
Then run `. $PROFILE` and `refreshworkspace`.

**`show`** — Print the generated function only; the user pastes it into their profile or script file, then runs `. $PROFILE` and `refreshworkspace`.

---

## How to Use

Run `refreshworkspace` once after setup, and again any time you add or remove repos:

```powershell
refreshworkspace                                    # rebuild the index
refreshworkspace -WorkspaceRoot E:\OtherWorkspace   # different root
refreshworkspace -OutputFile D:\my-repos.json       # custom output path
```

The AI agent reads the index file directly — no more folder scanning.
